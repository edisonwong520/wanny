# 关怀页面重设计

## 概述

将现有的关怀中心页面（CareCenterPage）和规则管理页面（CareRulesPage）合并为单一的三栏布局页面，提升信息密度和操作效率。

## 设计目标

1. **信息密度提升**：天气、建议、规则同屏展示，减少页面跳转
2. **操作效率优化**：常用操作（巡检、配置数据源）触手可及
3. **界面简洁**：移除冗余标题和小字，专注核心内容

## 组件拆分

### 左栏：天气面板 (WeatherPanel.vue)

**功能**：
- 显示实时天气（温度、天气状况、位置、数据源）
- 显示天气指数（湿度、体感、风向、气压）
- 数据源配置弹窗入口
- 刷新天气按钮

**交互**：
- 点击"数据源"按钮 → 弹出数据源配置弹窗
- 点击"刷新"按钮 → 刷新天气数据

**数据源配置弹窗**：
- 数据源类型选择（和风天气 / Open-Meteo / HA 实体）
- 根据类型显示不同配置表单
- 保存/取消操作

### 中栏：建议管理 (SuggestionsPanel.vue)

**功能**：
- 状态筛选器（待处理/已采纳/已执行/全部）
- 建议列表（按优先级颜色区分）
- 建议统计（待处理数量）
- 立即巡检按钮

**交互**：
- 点击筛选标签 → 过滤建议列表
- 点击建议卡片 → 展开详情/执行操作
- 点击"立即巡检" → 触发巡检扫描

### 右栏：规则管理 (RulesPanel.vue)

**功能**：
- 系统规则列表（只读，可启用/禁用）
- 自定义规则列表（可编辑、删除）
- 新建规则按钮

**交互**：
- 点击规则 → 编辑规则（弹窗或内联）
- 点击启用/禁用 → 切换规则状态
- 点击"新建" → 新建规则弹窗

## 数据源配置弹窗

**结构**：
```
┌─────────────────────────────────┐
│ 天气数据源配置              [✕] │
├─────────────────────────────────┤
│ 数据源类型                      │
│ [和风天气] [Open-Meteo] [HA实体]│
│                                 │
│ 名称                            │
│ [和风天气________________]      │
│                                 │
│ 城市ID (和风天气)               │
│ [101020100______________]       │
│                                 │
│ 或 经纬度 (Open-Meteo)          │
│ [纬度____] [经度____]           │
│                                 │
│ HA实体ID (HA实体)               │
│ [weather.home___________]       │
│                                 │
│   [取消]          [保存]        │
└─────────────────────────────────┘
```

**字段**：
- 和风天气：名称、城市ID、API Key
- Open-Meteo：名称、纬度、经度、时区
- HA 实体：名称、实体ID
