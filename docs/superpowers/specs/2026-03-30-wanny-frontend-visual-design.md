# Wanny Frontend 视觉设计规范 v1.0

## 1. 设计定位

Wanny（WeChat + Nanny）的视觉方向不是传统极客控制台，也不是纯消费级营销页。

它需要同时满足两种气质：

- **微信式易用性**：让用户有熟悉感、低门槛、低压迫感
- **AI 管家的科技感**：让人感觉系统是“活的”，但不过度炫技

因此，默认视觉应采用：

- 浅色主题
- 极简克制
- 局部光晕
- 清晰层级

## 2. 核心色板

### 2.1 主色与中性色

- **品牌主色 / Primary Accent**：`#07C160`
  - 用于关键操作按钮、核心状态、主要图表高亮、AI 激活提示
- **主背景 / Main Background**：`#F7F7F7`
  - 用于页面底层背景
- **卡片 / Surface**：`#FFFFFF`
  - 用于面板、表格容器、浮层、卡片
- **一级文本 / Primary Text**：`#191919`
  - 用于标题和主要内容
- **二级文本 / Secondary Text**：`#999999`
  - 用于提示信息、标签说明、时间戳、辅助文案
- **AI 光晕辅助色 / AI Glow Accent**：`#E1F8EA`
  - 用于 AI 正在思考、运行中、局部背景发光和轻毛玻璃衬底

## 3. 配色使用规则

### 3.1 背景

- 页面背景始终优先使用 `#F7F7F7`
- 不使用大面积深色底
- 可叠加非常淡的绿色径向光晕作为环境层，而不是作为内容层

### 3.2 面板

- 卡片统一使用白色或高透明白
- 阴影保持轻，不使用沉重阴影
- 边框优先使用极淡的黑色透明度边框，而不是重描边

### 3.3 强调色

- 绿色只用于重点，不做全局铺满
- 一个区域里最多保留一个主强调动作
- 高亮状态应以“绿底 + 轻阴影”或“浅绿底 + 深绿字”为主

### 3.4 文本层级

- 标题和核心数据统一使用 `#191919`
- 说明文字、占位、标签说明使用 `#999999`
- 不使用纯黑 `#000000`

## 4. AI 科技感表达方式

Wanny 的科技感不应该来自赛博朋克霓虹，而应该来自“轻微动态的秩序感”。

推荐方式：

- 在页面背景局部加入 `#E1F8EA` 的模糊径向光晕
- 在 AI 运行面板中加入浅绿色背景洗光
- 用轻微的漂浮、呼吸式动画表达运行中状态
- 用毛玻璃与磨砂卡片制造“系统正在工作”的精致感

禁止方式：

- 大面积高亮发光
- 大范围蓝紫赛博渐变
- 深色背景上强荧光色叠加

## 5. Console 特定规则

Console 是运营中枢，视觉上必须优先保证高信噪比。

因此：

- 信息密度可以高，但层级必须清晰
- 卡片之间通过浅阴影与留白分隔，而不是靠深色块区分
- 状态面板、待审批任务、设备策略矩阵等核心信息必须优先进入首屏
- 所有关键动作按钮都应有明确主次，不允许出现多个视觉上同等强度的主按钮

## 6. Landing Page 特定规则

Landing Page 后续用于推广产品，视觉上可以比 Console 更“讲故事”，但仍需遵守本规范：

- 仍以浅色为主
- 允许更大的留白与更柔和的光效
- 品牌绿用于建立微信熟悉感
- AI 光晕用于表达“Jarvis 在工作”

## 7. 动效规则

- 动效应慢、轻、柔和
- 推荐使用 200ms - 400ms 的 hover / press 过渡
- 背景光晕动画应低频，不可影响阅读
- 列表、卡片进入动画以轻微位移和透明度变化为主

## 8. 当前实现落地

当前前端已按本规范做了第一轮主题落地：

- Console 改为浅色主题
- 品牌主色切换为微信绿 `#07C160`
- 背景改为浅灰白 `#F7F7F7`
- 面板统一为白色卡片
- AI 光晕使用 `#E1F8EA` 构建局部动态背景

后续新增页面与组件必须继续遵守本规范，避免再回到深色主视觉或赛博朋克风格。
