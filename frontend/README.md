# Wanny Frontend

Wanny 的前端工作区负责两类界面：

- Landing Page：用于展示 Jarvis 的能力、入口与整体定位
- Jarvis Console：用于管理设备、任务、记忆、审批流与运行状态

## 技术栈

- Vue 3
- Vite
- TypeScript
- Tailwind CSS
- shadcn-vue
- vue-i18n

## 运行方式

建议使用现代 LTS Node 版本，推荐 `24.11.1`，兼容 `22+`。

```bash
npm install
npm run dev
```

## 目录说明

```text
frontend/
├── src/
│   ├── components/      # 通用界面组件与 UI 基础组件
│   ├── i18n/            # 中英文文案
│   ├── lib/             # 工具函数
│   ├── pages/           # 页面级视图（Landing / Console）
│   └── router/          # 前端路由
├── components.json      # shadcn-vue 配置
├── tailwind.config.ts   # Tailwind 样式配置
└── vite.config.ts       # Vite 配置
```

## 授权页复盘

平台授权这页之前设计得差，核心不是样式问题，而是信息边界错了：

- 把后端内部状态直接堆到前端：例如“已保存字段”“暂无已保存凭证字段”“最近更新时间”这类字段更适合排障或后台审计，不应该出现在普通授权流程里。
- 把一个简单动作做成了信息面板：用户要的只是绑定和解绑，但页面把状态说明、会话文案、字段列表、更新时间都塞进去了，主次颠倒。
- 弹窗没有聚焦单一任务：授权弹窗应该只承载完成授权所需的信息，例如二维码和可点击链接；其余说明文字会分散注意力。
- 组件结构没有先约束滚动区域：把滚动直接放在外层弹窗容器上，会导致滚动条顶出圆角边界，视觉上很脏。

后续约束：

- 平台授权列表只展示平台、连接状态、绑定/解绑动作。
- 授权弹窗只展示完成当前授权所必需的元素。
- 排障信息默认不进入主界面，除非明确做成诊断视图。
- 任何带圆角的弹窗，滚动都必须放在内层内容区，外层容器负责裁切。
